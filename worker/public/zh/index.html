功能

不會阻礙你的 RAG 後端

監看、解析、嵌入並提供服務 — 單一 Go 二進位檔，無需外部向量資料庫。

自動擷取

將檔案放入監看資料夾。KuraDB 會自動辨識、解析、分塊並排入嵌入佇列 — 無需任何指令或結構定義。

語意向量搜尋

兩階段的餘弦搜尋：先進行粗略的來源層級排序，再於各 CPU 核心間平行掃描片段層級。自行實作，不依賴 ToriiDB。

CJK 關鍵字搜尋

以 gse 驅動的中文斷詞並搭配停用詞。根據關鍵字命中次數進行計分，實現精準、具語言感知的比對。

唯讀合約

HTTP API 刻意未設計任何變更端點。資料只由監看管線單向進入 — 這是信任邊界，而非設定旗標。

以 SQLite 為事實來源

每個片段與向量都存放在 SQLite 中。記憶體中的向量快取可重建 — 永遠不是權威來源，重啟後也不會遺失。

智慧重新嵌入

只在內容確實變更時才使向量失效。相同檔案會保留既有嵌入 — 不會浪費 OpenAI 的來回通訊。

查詢快取

查詢嵌入會快取在全域 SQLite 儲存庫中。重複搜尋會跳過嵌入呼叫，直接進入向量掃描。

軟刪除一致性

移除的檔案會被排除而非真正刪除。所有語意與關鍵字查詢都會嚴格過濾已排除的內容。

原生支援 Agenvoy

綁定隨機本機埠並將 URL 發佈到端點檔案，讓 Agenvoy 能以子程序形式自動發現並使用。

搜尋

兩種方式找到任何內容

語意負責含義，關鍵字負責精確詞語。兩者都會過濾已排除的內容，且絕不洩漏內部計分欄位。

介面

唯讀 API 與輕量 CLI

四個端點供查詢，四個子命令供管理。完全沒有任何變更表面。

HTTP Endpoints

所有端點都在 /api 下，綁定到 localhost 上的隨機空閒埠。limit 預設為 10，最大值 100。

GET/api/health
GET/api/list
GET/api/semantic?db=&q=&limit=
GET/api/keyword?db=&q=&limit=

CLI Commands

不帶任何參數執行 kura 即可啟動伺服器。子命令用來管理資料庫登錄。

kura · start server kura add <name> kura list kura remove <name> kura edit <old> <new> kura help

快速開始

四個步驟，從資料夾到搜尋

安裝一次、註冊一個資料庫、放入你的檔案，然後開始查詢。

1

安裝

自動佈署 Go 並建置 kura 二進位檔。支援 macOS 與 Linux。

curl -fsSL https://kuradb.agenvoy.com/scripts/install.sh | bash
2

新增資料庫

建立收件匣與 ~/Kura_notes 符號連結。

kura add notes
3

放入檔案

將文件複製到已建立符號連結的收件匣資料夾。

cp ~/Documents/*.pdf ~/Kura_notes/
4

啟動並查詢

伺服器會自動載入、監看並嵌入。

kura
